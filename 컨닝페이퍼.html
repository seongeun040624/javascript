margin 태그와 padding 태그가 작동하는 방식이 다르므로 차이점을 잘 알아둘 것

* nav = 페이지를 이동시키는 내비게이션 메뉴를 구성할 때

* section = 아래 article과 aside를 합친 영역

* article = 바디의 주요 내용이자 본문

* aside = 바디의 주요 내용 외에 기타 내용이나 첨부 사진 등등

* footer = 하단 메뉴 요소나 저작권 등

내용 기재
더 자세한 내용은 클릭하여 확인할 수 있다.

* text-align: center; = 텍스트의 정렬 방법을 지정한다. center인 경우 중앙정렬이 된다.

* line-height : = 해당 요소의 자간을 조정한다. 요소의 높이만큼 line-height을 주면 텍스트가 수직중간정렬이 된다. 그 후 text-align: center; 을 사용함으로서 수평중간정렬도 가능하다.

* letter-spacing : = 해당 요소의 자간을 조정한다.

* word-spacing : = 단어간의 간격을 조정한다. 띄어쓰기와 ,로 단어를 구분한다.

* text-indent : = 입력한 값 만큼 들여쓰기가 적용된다.

* border : = 요소에 테두리를 형성한다. {border:10px solid color;} 이런 방법으로 작성되는데 {box-sizing: content-box;}를 함께 사용하면 테두리나 마진, 패딩 너비는 앞서 지정한 요소 사이즈 안에서 형성된다. 겹친 border를 제거하기위해 border-collapse를 생각할 수 있지만 이 태그는 table태그 안에서만 작동한다. border에 또 border를 주는 방법은 outline: 1px solid; 를 사용하는 것이다.

* width: fit-content; = 블록 요소 너비를 내부 컨텐츠 내용 만큼으로 강제로 맞출 수 있다.

컨텐츠 없이 인접한 부모-자식 요소는 마진병합현상이 발생하기도 하는데 이를 해결하기 위해서는 한쪽 요소에 padding-top: 1px; margin-top: -1px; 값으로 디폴트를 정해준 다음 다른 쪽에 원하는 마진 값을 주면 된다.

자세한 내용은 클릭에서 확인할 수 있다.

선택자

보통은 원하는 요소를 선택하기위해 #id값이나 .class 값을 사용하지만, 많은 목록 중에 하나의 p나 li 등(순서를 가지는 요소) 일회성의 요소를 선택할 때 다음과 같이 child 태그나 after, before를 사용할 수 있다.

* :nth-child(){} = ()~의 몇 번 째 요소로서 사용된다.

▲ :nth-child(even)이면 짝수번 째, :nth-child(odd)이면 홀수번 째 요소 선택이 가능하다.
▲ :nth-child(2n)이면 2의 배수 번째로 선택한다.

* :first-child{} = ~태그의 첫번째 요소로서 사용된다.

* :last-child{} = ~태그의 마지막 요소로서 사용된다.

* ~::before{content: ""} = ~태그의 바로 전에 ""내용을 추가할 뿐만 아니라 괄호에 따라오는 속성 값까지도 수정할 수 있다.

* ~::after{content: ""} = ~태그의 바로 뒤에 ""내용을 추가할 뿐만 아니라 괄호에 따라오는 속성 값까지도 수정할 수 있다.

* ~::first-line{} = ~태그의 첫 번째 줄 요소에만 {}속성 값을 적용한다.

주의점 : 그룹 선택자, 다중 선택을 하기위해 #header h1,h2 이렇게 작성하면 h2는 일반적인 h2를 선택한다. 헤더 후손을 선택하기 위해선 #header h1,#header h2로 작성해야한다.


속성 선택자

여러개의 요소 중 특정 키워드가 포함된 요소를 선택할 수 있다. 이는 제이쿼리(jQuery) 선택자 문법과 유사하다.

* 태그, id, class [title^=제과]{} : 해당 값으로 시작하는 요소

* 태그, id, class [title$=기능장]{} : 해당 값으로 끝나는 요소

* 태그, id, class [title*=산업기사]{} : 해당 값이 포함된 요소
